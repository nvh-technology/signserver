Deployment Commands for SignServer:

1. Install Composer dependencies:
   composer install

2. Run database migrations (this will create all necessary tables, including roles and permissions tables):
   php artisan migrate

3. Seed roles and permissions (this will create 'admin' and 'user' roles, define permissions, and assign the 'admin' role to the user with ID 1):
   php artisan db:seed --class=RolesAndPermissionsSeeder

4. Clear application caches:
   php artisan optimize:clear

Note: The `2025_08_13_174559_create_permission_tables.php` migration is a core migration generated by the spatie/laravel-permission package and should remain as a single file for proper package functionality. The roles and permissions creation logic is already separated into the `RolesAndPermissionsSeeder`.

To ensure a fully privileged admin account, make sure a user with ID 1 exists in your database before running the seeder. The `RolesAndPermissionsSeeder` will attempt to assign the 'admin' role to this user.
